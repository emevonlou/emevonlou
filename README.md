# About Me

Whoever studies systems deeply must resist becoming merely one of their components.

I am driven by a disciplined curiosity:  
to understand how systems behave under pressure, how they fail, and what they reveal when carefully observed.

I do not approach technology as a collection of tools.  
I approach it as a structure of logic, constraints, and consequences.

---

## Focus and Practice

My work sits at the intersection of:

- Web development, with a growing focus on backend fundamentals  
- Security-aware thinking applied to everyday code  
- System observation beyond happy paths  
- Understanding why something works and how it can break  

I am particularly interested in:
- Hidden attack surfaces in seemingly simple implementations  
- Trust assumptions embedded in software design  
- The human layer behind technical decisions  

Security, to me, is not paranoia. It is clarity and method.

## How I Think About Code

- Code is an argument, not decoration  
- Complexity must justify itself  
- Automation without understanding creates fragile systems  
- Security begins at design, not at patch time  

I read code as one analyzes a system:  
looking for assumptions, shortcuts, and silent risks.
